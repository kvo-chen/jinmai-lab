## 问题判断

* 你截图中的日志显示 Next.js 的 `src/app/*` 与 `next/image` 的 ESLint 提示，这与我们当前 Vite 项目不符，说明域名可能绑定到了“另一个 Next 项目”的部署，而不是本项目。

* 同时此前 Vercel 对输出目录的期望为 `dist`，我们已经在 `vercel.json` 指定了 `outputDirectory: dist/static`。若仍异常，可能是项目设置或重写规则导致路径错配。

## 修复方案

1. **确认域名绑定到正确项目**

   * 在 Vercel 控制台 → `Settings → Domains`，确保 `jinmai-lab.tech` 与 `www.jinmai-lab.tech` 只绑定到当前项目。

   * 若被其他项目占用，移除它们的绑定，然后在当前项目重新添加。

2. **项目构建设置（Vercel 控制台）**

   * Framework Preset：选择 `Vite` 或 `Other`（不要选择 Next.js）

   * Install Command：`pnpm install`

   * Build Command：`pnpm build`

   * Output Directory：`dist/static`

   * Node 版本：`18` 或 `20`

3. **vercel.json 简化与修正**

   * 移除 `rewrites` 到 `/static/$1`（当前 outputDirectory 已是 `dist/static`，静态文件会按根路径部署，不需要重写）

   * 保留首页 `Cache-Control: no-cache` 头，避免首次展示旧缓存

   * 保持 `functions` 为空（纯静态部署），避免被识别为 Serverless 项目

4. **忽略后端目录**

   * `.vercelignore` 保持忽略 `api/`（纯静态场景），确保不会触发 Next/Node 路由检测

5. **重新部署（无缓存）**

   * 在 Vercel 控制台 → `Deploys` → 选择最近一次 → `Redeploy` → 选 `Rebuild without cache`

6. **验证**

   * 访问 `https://jinmai-lab.tech/?v=<timestamp>` 绕过浏览器缓存

   * 检查：

     * 页脚版本标识显示（我们已在 `Home.tsx` 增加）

     * 管理员登录自动跳转 `/admin`

     * 后台四个功能页正常

     * 响应头首次 `X-Vercel-Cache: MISS`，随后 `HIT`
